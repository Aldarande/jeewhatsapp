JeeWhatsApp BETA

Intégration WhatsApp bidirectionnelle pour Jeedom via Baileys.
Canal de communication dédié — 100 % local, aucune donnée chez un tiers.

⚠️ Version bêta Cette documentation décrit la version bêta du plugin. Les fonctionnalités en cours de développement peuvent changer avant la sortie de la version stable. Pour la version stable, consultez la documentation principale.

Présentation

JeeWhatsApp connecte votre installation Jeedom à WhatsApp via Baileys, une bibliothèque open-source qui établit une connexion WebSocket directe avec WhatsApp Web. Aucun compte tiers, aucune clé API, aucun abonnement — tout se passe entre votre serveur Jeedom et les serveurs WhatsApp.

Ce que le plugin ne fait pas (actuellement)

  • Envoi de médias (images, vidéos, documents, vocaux)
  • Réception de messages hors du groupe canal
  • Envoi vers un numéro non enregistré sur WhatsApp
  • Notifications push pour le propriétaire du compte — voir limitation ci-dessous
⚠️ Limitation — notifications push JeeWhatsApp utilise votre propre compte WhatsApp (lié par QR code). WhatsApp ne notifie jamais le propriétaire du compte pour ses propres messages — cette règle s'applique aussi aux mentions (@vous), testées et confirmées sans effet.

Conséquence : quand Jeedom poste dans le groupe canal, vous ne recevez aucune notification push Les messages sont visibles en ouvrant le groupe, mais pas d'alerte sonore ni de bannière.

Solution : un deuxième compte WhatsApp dédié à Jeedom. Un numéro virtuel (Google Voice, eSIM bas de gamme…) connecté en tant que "bot Jeedom", membre du groupe canal — votre compte principal reçoit alors les messages comme n'importe quel message de groupe, avec notification normale.

Principe — le groupe canal

JeeWhatsApp repose sur un groupe WhatsApp dédié qui sert de canal de communication bidirectionnel entre Jeedom et vous.

🏠
Jeedom
⇄groupe canal
👥
Groupe WhatsApp
"jeewhatsapp"
⇄notifications
📱
Votre téléphone
  • Jeedom → groupe : chaque alerte ou rapport envoyé depuis un scénario apparaît dans le groupe, préfixé (ex : 🏠 Mouvement détecté !)
  • Groupe → Jeedom : vos messages dans le groupe sont reçus par Jeedom et déclenchent vos scénarios ou le moteur d'interactions
  • Les messages en dehors du groupe (messages directs, autres groupes) sont ignorés
  • Vous recevez une notification WhatsApp comme pour n'importe quel message de groupe
💡 Préfixe Jeedom Tous les messages envoyés par Jeedom sont automatiquement préfixés (ex : 🏠 ). Cela vous permet de distinguer visuellement les alertes Jeedom de vos propres messages dans le groupe. Le daemon ignore ses propres messages (fromMe) pour éviter les boucles.

Prérequis

  • Jeedom 4.4 ou supérieur
  • Node.js 18 ou supérieur installé sur la machine Jeedom
  • Un smartphone avec l'application WhatsApp pour scanner le QR code
ℹ️ Vérifier Node.js Dans un terminal sur votre machine Jeedom : node --version
La version affichée doit être v18.x ou supérieure.

Installation

  1. 1
    Installer le plugin
    Copiez le dossier jeewhatsapp/ dans le répertoire plugins/ de votre Jeedom, puis activez-le dans Plugins → Gestion des plugins.
  2. 2
    Installer les dépendances
    Sur la page du plugin, cliquez sur Installer les dépendances. Le script installe @whiskeysockets/baileys, qrcode et pino via npm.
    Cette étape peut prendre 2 à 5 minutes selon votre connexion internet.
  3. 3
    Démarrer le daemon
    Dans la page de configuration du plugin, cliquez sur Démarrer le daemon. Le statut doit passer à OK en quelques secondes.
  4. 4
    Créer un équipement et scanner le QR code
    Créez un équipement, configurez le groupe canal, sauvegardez, puis rendez-vous sur l'onglet Connexion WhatsApp.

Configuration de l'équipement

Allez dans Plugins → Communication → JeeWhatsApp et cliquez sur Ajouter.

Lier le groupe canal

Après la sauvegarde, vous devez lier un groupe WhatsApp à l'équipement. Vous devez d'abord être connecté à WhatsApp (QR code scanné).

Option A — Groupe existant

  1. 1
    Créez un groupe WhatsApp depuis votre téléphone et nommez-le (ex : jeewhatsapp).
  2. 2
    Dans la page équipement, renseignez ce nom dans le champ Groupe canal.
  3. 3
    Cliquez sur Rechercher — le champ Groupe lié se remplit automatiquement avec le JID.
  4. 4
    Sauvegardez l'équipement.

Option B — Créer le groupe depuis Jeedom

  1. 1
    Renseignez le nom souhaité dans le champ Groupe canal.
  2. 2
    Cliquez sur Créer — le groupe est créé sur WhatsApp et le JID est renseigné.
  3. 3
    Sauvegardez l'équipement.
  4. 4
    Depuis votre téléphone, ajoutez les membres souhaités au groupe.
ℹ️ Le champ "Groupe lié" (JID) Ce champ en lecture seule contient l'identifiant technique du groupe WhatsApp (format 120363…@g.us). Il est renseigné automatiquement et ne doit pas être modifié manuellement. Le daemon le mémorise dans auth/{id}/group_jid.txt pour accélérer les redémarrages.
💡 Recherche automatique au démarrage Dès que la connexion WhatsApp est établie, le daemon cherche automatiquement le groupe par son nom. Le résultat apparaît dans les logs jeewhatsapp : ✓ Groupe "jeewhatsapp" → 120363…@g.us.

Connexion par QR code

Rendez-vous sur l'onglet Connexion WhatsApp de la page équipement.

  1. 1
    Un QR code s'affiche automatiquement (rafraîchissement toutes les 8 secondes). Si le daemon vient d'être démarré, patientez quelques secondes.
  2. 2
    Ouvrez WhatsApp sur votre téléphone → Paramètres → Appareils liés → Lier un appareil.
  3. 3
    Scannez le QR code affiché dans Jeedom.
  4. 4
    Le statut passe à ✅ Connecté. Le daemon recherche alors automatiquement le groupe canal.
💡 Session persistante Une fois connecté, les credentials sont sauvegardés dans resources/jeewhatsappd/auth/{id}/. Vous n'aurez pas à rescanner le QR code à chaque redémarrage du daemon.
ℹ️ Déconnexion par WhatsApp Si WhatsApp révoque l'accès (rare), la session est supprimée automatiquement et un nouveau QR code apparaît dans l'onglet. Rescannez simplement.

💡 Champ "Titre" — override optionnel Jeedom affiche deux champs pour les commandes de type message : Titre et Message.
Dans JeeWhatsApp, le champ Titre est un override optionnel :
— Vide → envoi dans le groupe canal (comportement par défaut)
— Numéro (ex : 33612345678) → envoi direct à ce numéro
— JID de groupe (ex : 120363…@g.us) → envoi dans ce groupe spécifique

Interactions Jeedom

Lorsque l'option Interactions Jeedom est activée, chaque message reçu dans le groupe canal est soumis au moteur d'interactions Jeedom. Si une interaction correspond, la réponse est automatiquement envoyée dans le groupe canal.

Message reçu dans le groupeRéponse automatique de Jeedom
température salon🏠 La température du salon est de 21°C
allume la lumière🏠 Lumière allumée
statut alarme🏠 Alarme désactivée
ℹ️ Configurer les interactions Les interactions se configurent dans Outils → Interactions dans Jeedom. Elles fonctionnent avec les commandes de tous vos équipements.
⚠️ Réponses publiques dans le groupe Les réponses automatiques sont envoyées dans le groupe, visibles de tous les membres. Si le groupe contient plusieurs personnes, tous verront les réponses de Jeedom.

Dépannage

Je ne reçois pas de notification WhatsApp quand Jeedom envoie un message
  • C'est une limitation de WhatsApp : un compte ne reçoit jamais de notification pour ses propres messages
  • Les mentions @vous ne contournent pas cette règle — testé et confirmé
  • Seule solution : utiliser un deuxième compte WhatsApp dédié à Jeedom (numéro virtuel ou SIM dédiée), membre du groupe canal
  • Votre compte principal reçoit alors les messages de Jeedom comme n'importe quel message de groupe, avec notification normale
Le daemon ne démarre pas
  • Vérifiez que Node.js 18+ est installé : node --version
  • Relancez l'installation des dépendances depuis la page du plugin
  • Consultez le log jeewhatsapp dans Analyse → Logs
  • Vérifiez que le port 55148 (port par défaut) n'est pas déjà utilisé par un autre service — si c'est le cas, changez-le dans Plugins → JeeWhatsApp → Configuration
Le QR code ne s'affiche pas
  • Vérifiez que le daemon est démarré (statut OK dans la gestion du plugin)
  • Redémarrez le daemon puis rafraîchissez la page équipement
  • Consultez le log jeewhatsapp pour détecter une erreur Baileys
  • Si une session existe déjà, le QR code n'apparaît pas — le statut doit être Connecté
Le groupe canal n'est pas trouvé
  • Vérifiez que le nom dans Groupe canal correspond exactement au nom du groupe WhatsApp (sensible à la casse)
  • Le groupe doit exister sur WhatsApp et votre compte doit en être membre
  • Cliquez sur Rechercher dans la page de configuration pour forcer la recherche
  • Consultez le log jeewhatsapp : le daemon affiche Groupe "xxx" introuvable avec le nom recherché
Les messages ne sont pas reçus
  • Vérifiez que le statut dans l'onglet Connexion WhatsApp affiche Connecté
  • Vérifiez que le champ Groupe lié est renseigné
  • Seuls les messages texte du groupe canal sont traités — les médias sont ignorés
  • Consultez le log jeewhatsapp pour voir si le daemon reçoit bien les messages
L'envoi échoue
  • Vérifiez que le statut WhatsApp est Connecté
  • Vérifiez que le champ Groupe lié est renseigné
  • Testez depuis l'onglet Test (laissez Destinataire vide pour envoyer dans le groupe)
  • Consultez le log jeewhatsapp
WhatsApp a déconnecté la session (logout)
  • Le daemon détecte la déconnexion et supprime les credentials automatiquement
  • Un nouveau QR code apparaît dans l'onglet Connexion WhatsApp
  • Rescannez le QR code — à la reconnexion, le groupe canal est automatiquement retrouvé

Architecture technique

Groupe WhatsApp "jeewhatsapp" │ │══════════ WebSocket Baileys (connexion directe) ══════════│ jeewhatsappd.js │ │── messages.upsert ──────────────────────────────────────▶ │ │ ├─ filtre : remoteJid === groupJid ? → sinon ignoré │ ├─ filtre : fromMe ? → ignoré (message de Jeedom) │ └─ POST callback.php?apikey= │ └─ jeewhatsapp::callback() │ └─ cmd.event(last_message, last_sender…) │ │◀── sock.sendMessage(groupJid, { text: préfixe + message }) │ POST 127.0.0.1:55148/action {"action":"send",…} │ ← scénario → cmd::execute() → sendToDaemon()

Composants

ComposantTechnologieRôle
jeewhatsappd.js Node.js (ESM) Daemon — connexion Baileys, filtre groupe canal, serveur HTTP local port 55148
jeewhatsapp.class.php PHP Logique Jeedom : lifecycle daemon, commandes, envoi groupe
callback.php PHP Endpoint HTTP local appelé par le daemon à chaque message reçu dans le groupe
jeewhatsapp.ajax.php PHP Actions AJAX : test d'envoi, QR code, findGroup, createGroup, statut

FichierContenu
