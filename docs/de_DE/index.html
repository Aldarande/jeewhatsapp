JeeWhatsApp Self-hosted

Bidirektionale WhatsApp-Integration für Jeedom über Baileys.
Ein dedizierter Kommunikationskanal — 100 % lokal, keine Daten bei Dritten.

Übersicht

JeeWhatsApp verbindet Ihre Jeedom-Installation mit WhatsApp über Baileys, eine Open-Source-Bibliothek, die eine direkte WebSocket-Verbindung zu WhatsApp Web aufbaut. Kein Drittanbieter-Konto, kein API-Schlüssel, kein Abonnement — alles läuft zwischen Ihrem Jeedom-Server und den WhatsApp-Servern.

⚠️ Inoffizielle Anwendung — Sperrrisiko JeeWhatsApp verwendet Baileys, eine Bibliothek, die WhatsApp Web emuliert. Dieses Plugin ist nicht mit Meta verbunden und nutzt nicht die offizielle WhatsApp-Business-API. Übermäßige oder robotergestützte Nutzung kann zur vorübergehenden oder dauerhaften Sperrung des verwendeten WhatsApp-Kontos durch Meta führen.

✅ Anwendungen mit geringem Risiko (empfohlen):
  • Einige Heimautomatisierungs-Benachrichtigungen pro Tag in eine private Gruppe senden (Bewegung, Temperatur, Türöffnung …)
  • Über Interaktionen auf Ihre eigenen Nachrichten in der Kanalgruppe antworten
  • Das Plugin mit einem seit Langem aktiven, persönlichen WhatsApp-Konto verwenden
  • Nachrichten an eine Gruppe senden, in der Sie Mitglied sind, mit bekannten Kontakten
❌ Zu vermeidende Anwendungen:
  • Hunderte Nachrichten pro Tag automatisch senden
  • Nachrichten an Unbekannte oder an nicht in Ihren Kontakten gespeicherte Nummern senden
  • Ein neues WhatsApp-Konto ohne Verlauf menschlicher Konversationen verwenden
  • Nachrichten mit perfekt regelmäßigem Timing senden (alle X Sekunden)

Was das Plugin (derzeit) nicht tut

  • Senden von Medien (Bilder, Videos, Dokumente, Sprachnachrichten)
  • Empfang von Nachrichten außerhalb der Kanalgruppe
  • Senden an eine nicht bei WhatsApp registrierte Nummer
  • Push-Benachrichtigungen für den Kontoinhaber — siehe Einschränkung unten
⚠️ Einschränkung — Push-Benachrichtigungen JeeWhatsApp verwendet Ihr eigenes WhatsApp-Konto (per QR-Code verknüpft). WhatsApp benachrichtigt den Kontoinhaber niemals über seine eigenen Nachrichten — diese Regel gilt auch für Erwähnungen (@sie), getestet und ohne Wirkung bestätigt.

Folge: Wenn Jeedom in die Kanalgruppe postet, erhalten Sie keine Push-Benachrichtigung. Die Nachrichten sind sichtbar, wenn Sie die Gruppe öffnen, aber es gibt keinen Tonalarm oder Banner.

Lösung: ein zweites WhatsApp-Konto, das Jeedom gewidmet ist. Eine virtuelle Nummer (Google Voice, eine günstige eSIM …), die als „Jeedom-Bot" verbunden ist, Mitglied der Kanalgruppe — Ihr Hauptkonto erhält die Nachrichten dann wie jede andere Gruppennachricht, mit normaler Benachrichtigung.

Prinzip — die Kanalgruppe

JeeWhatsApp basiert auf einer dedizierten WhatsApp-Gruppe, die als bidirektionaler Kommunikationskanal zwischen Jeedom und Ihnen dient.

🏠
Jeedom
⇄Kanalgruppe
👥
WhatsApp-Gruppe
"jeewhatsapp"
⇄Benachrichtigungen
📱
Ihr Telefon
  • Jeedom → Gruppe: jede aus einem Szenario gesendete Warnung oder jeder Bericht erscheint in der Gruppe, mit Präfix (z. B. 🏠 Bewegung erkannt!)
  • Gruppe → Jeedom: Ihre Nachrichten in der Gruppe werden von Jeedom empfangen und lösen Ihre Szenarien oder die Interaktions-Engine aus
  • Nachrichten außerhalb der Gruppe (Direktnachrichten, andere Gruppen) werden ignoriert
  • Sie erhalten eine WhatsApp-Benachrichtigung wie bei jeder anderen Gruppennachricht
💡 Jeedom-Präfix Alle von Jeedom gesendeten Nachrichten erhalten automatisch ein Präfix (z. B. 🏠 ). So können Sie Jeedom-Warnungen visuell von Ihren eigenen Nachrichten in der Gruppe unterscheiden. Der Daemon ignoriert seine eigenen Nachrichten (fromMe), um Schleifen zu vermeiden.

Voraussetzungen

  • Jeedom 4.4 oder höher
  • Node.js 18 oder höher auf dem Jeedom-Rechner installiert
  • Ein Smartphone mit der WhatsApp-App zum Scannen des QR-Codes
ℹ️ Node.js prüfen In einem Terminal auf Ihrem Jeedom-Rechner: node --version
Die angezeigte Version muss v18.x oder höher sein.

Installation

  1. 1
    Plugin installieren
    Kopieren Sie den Ordner jeewhatsapp/ in das Verzeichnis plugins/ Ihrer Jeedom-Installation und aktivieren Sie ihn unter Plugins → Plugin-Verwaltung.
  2. 2
    Abhängigkeiten installieren
    Klicken Sie auf der Plugin-Seite auf Abhängigkeiten installieren. Das Skript installiert @whiskeysockets/baileys, qrcode und pino über npm.
    Dieser Schritt kann je nach Internetverbindung 2 bis 5 Minuten dauern.
  3. 3
    Daemon starten
    Klicken Sie auf der Plugin-Konfigurationsseite auf Daemon starten. Der Status sollte innerhalb weniger Sekunden auf OK wechseln.
  4. 4
    Ein Gerät erstellen und den QR-Code scannen
    Erstellen Sie ein Gerät, konfigurieren Sie die Kanalgruppe, speichern Sie und wechseln Sie dann zum Tab WhatsApp-Verbindung.

Kanalgruppe verknüpfen

Nach dem Speichern müssen Sie eine WhatsApp-Gruppe mit dem Gerät verknüpfen. Sie müssen zuerst mit WhatsApp verbunden sein (QR-Code gescannt).

Option A — Bestehende Gruppe

  1. 1
    Erstellen Sie eine WhatsApp-Gruppe auf Ihrem Telefon und benennen Sie sie (z. B. jeewhatsapp).
  2. 2
    Geben Sie auf der Geräteseite diesen Namen im Feld Kanalgruppe ein.
  3. 3
    Klicken Sie auf Suchen — das Feld Verknüpfte Gruppe wird automatisch mit der JID ausgefüllt.
  4. 4
    Speichern Sie das Gerät.

Option B — Die Gruppe aus Jeedom erstellen

  1. 1
    Geben Sie den gewünschten Namen im Feld Kanalgruppe ein.
  2. 2
    Klicken Sie auf Erstellen — die Gruppe wird in WhatsApp erstellt und die JID ausgefüllt.
  3. 3
    Speichern Sie das Gerät.
  4. 4
    Fügen Sie von Ihrem Telefon aus die gewünschten Mitglieder zur Gruppe hinzu.
ℹ️ Das Feld „Verknüpfte Gruppe" (JID) Dieses schreibgeschützte Feld enthält die technische Kennung der WhatsApp-Gruppe (Format 120363…@g.us). Es wird automatisch ausgefüllt und darf nicht manuell geändert werden. Der Daemon speichert es in auth/{id}/group_jid.txt, um Neustarts zu beschleunigen.
💡 Automatische Suche beim Start Sobald die WhatsApp-Verbindung hergestellt ist, sucht der Daemon die Gruppe automatisch anhand ihres Namens. Das Ergebnis erscheint in den jeewhatsapp-Logs: ✓ Gruppe "jeewhatsapp" → 120363…@g.us.

Verbindung per QR-Code

Wechseln Sie zum Tab WhatsApp-Verbindung auf der Geräteseite.

  1. 1
    Ein QR-Code wird automatisch angezeigt (Aktualisierung alle 8 Sekunden). Wenn der Daemon gerade gestartet wurde, warten Sie einige Sekunden.
  2. 2
    Öffnen Sie WhatsApp auf Ihrem Telefon → Einstellungen → Verknüpfte Geräte → Gerät verknüpfen.
  3. 3
    Scannen Sie den in Jeedom angezeigten QR-Code.
  4. 4
    Der Status wechselt zu ✅ Verbunden. Der Daemon sucht dann automatisch nach der Kanalgruppe.
💡 Persistente Sitzung Nach dem Verbinden werden die Zugangsdaten in resources/jeewhatsappd/auth/{id}/ gespeichert. Sie müssen den QR-Code nicht bei jedem Daemon-Neustart erneut scannen.
ℹ️ Trennung durch WhatsApp Wenn WhatsApp den Zugriff widerruft (selten), wird die Sitzung automatisch gelöscht und ein neuer QR-Code erscheint im Tab. Einfach erneut scannen.

Jeedom-Interaktionen

Wenn die Option Jeedom-Interaktionen aktiviert ist, wird jede in der Kanalgruppe empfangene Nachricht an die Jeedom-Interaktions-Engine übergeben. Wenn eine Interaktion passt, wird die Antwort automatisch in die Kanalgruppe gesendet.

In der Gruppe empfangene NachrichtAutomatische Antwort von Jeedom
Temperatur Wohnzimmer🏠 Die Temperatur im Wohnzimmer beträgt 21°C
Licht einschalten🏠 Licht eingeschaltet
Alarmstatus🏠 Alarm deaktiviert
ℹ️ Interaktionen konfigurieren Interaktionen werden in Jeedom unter Werkzeuge → Interaktionen konfiguriert. Sie funktionieren mit den Befehlen all Ihrer Geräte.
⚠️ Öffentliche Antworten in der Gruppe Automatische Antworten werden in die Gruppe gesendet und sind für alle Mitglieder sichtbar. Wenn die Gruppe mehrere Personen enthält, sehen alle die Antworten von Jeedom.

Nachrichtenvorlagen

Im Tab Vorlagen können Sie wiederverwendbare Nachrichten mit Jeedom-Variablen definieren. Anstatt denselben Text in jedem Szenario neu zu schreiben, definieren Sie eine Vorlage einmal und rufen sie über ihren Schlüssel auf.

Format

Eine Vorlage pro Zeile, im Format schlüssel=Nachrichtentext:

willkommen=Willkommen zu Hause!
alarm=⚠️ Alarm: #[Haus][Melder][Anwesenheit]# !
nacht=Gute Nacht — automatisches Schließen der Rollläden
bericht=☀️ Wohnzimmer: #[Wohnzimmer][Temperatur][Wert]#°C — Tür: #[Eingang][Öffnung][Wert]#
# Zeilen, die mit # beginnen, sind Kommentare
  • Der Vorlagenschlüssel ist nicht case-sensitiv (Alarm = alarm)
  • Die Jeedom-Tags #[Objekt][Gerät][Befehl]# werden beim Senden automatisch aufgelöst. ⚠️ Die Namen von Objekt, Gerät und Befehl sind case-sensitiv (wie überall in Jeedom): sie müssen exakt übereinstimmen — #[Wohnzimmer][Temperatur][Wert]# ≠ #[wohnzimmer][temperatur][wert]#
  • Leere Zeilen und solche, die mit # beginnen, werden ignoriert

Verwendung in einem Szenario

Verwenden Sie den Aktionsbefehl Vorlage senden und geben Sie den Schlüssel im Feld Nachricht ein:

📋 Eine Vorlage aus einem Szenario senden

Aktion[Mein WhatsApp][Vorlage senden]
Nachrichtalarm (Vorlagenschlüssel)
Titel(leer = Kanalgruppe)
💡 Vorschau ohne Speichern Die Schaltfläche Vorlagen-Vorschau im Tab zeigt die Liste der erkannten Vorlagen in Echtzeit an, ohne speichern zu müssen — praktisch, um die Syntax vor dem Speichern zu prüfen.
ℹ️ Unterschied zu Nachricht senden Nachricht senden sendet den eingegebenen Text direkt.
Vorlage senden löst zuerst den Schlüssel auf, um den Vorlagentext abzurufen, und löst dann die Jeedom-Tags vor dem Senden auf. So können Sie Ihre Nachrichten zentralisieren und an einer einzigen Stelle ändern, ohne die Szenarien anzufassen.

Fehlerbehebung

Ich erhalte keine WhatsApp-Benachrichtigung, wenn Jeedom eine Nachricht sendet
  • Das ist eine Einschränkung von WhatsApp: ein Konto erhält niemals eine Benachrichtigung für seine eigenen Nachrichten
  • Erwähnungen @sie umgehen diese Regel nicht — getestet und bestätigt
  • Einzige Lösung: ein zweites WhatsApp-Konto verwenden, das Jeedom gewidmet ist (virtuelle Nummer oder dedizierte SIM), Mitglied der Kanalgruppe
  • Ihr Hauptkonto erhält die Nachrichten von Jeedom dann wie jede andere Gruppennachricht, mit normaler Benachrichtigung
Der Daemon startet nicht
  • Prüfen Sie, ob Node.js 18+ installiert ist: node --version
  • Starten Sie die Installation der Abhängigkeiten über die Plugin-Seite neu
  • Sehen Sie sich das Log jeewhatsapp unter Analyse → Logs an
  • Prüfen Sie, ob der Port 55148 (Standardport) nicht bereits von einem anderen Dienst belegt ist — falls doch, ändern Sie ihn unter Plugins → JeeWhatsApp → Konfiguration
Der QR-Code wird nicht angezeigt
  • Prüfen Sie, ob der Daemon gestartet ist (Status OK in der Plugin-Verwaltung)
  • Starten Sie den Daemon neu und aktualisieren Sie die Geräteseite
  • Sehen Sie sich das Log jeewhatsapp an, um einen Baileys-Fehler zu erkennen
  • Wenn bereits eine Sitzung existiert, erscheint der QR-Code nicht — der Status muss Verbunden sein
Die Kanalgruppe wird nicht gefunden
  • Prüfen Sie, ob der Name in Kanalgruppe exakt dem Namen der WhatsApp-Gruppe entspricht (case-sensitiv)
  • Die Gruppe muss in WhatsApp existieren und Ihr Konto muss Mitglied sein
  • Klicken Sie auf der Konfigurationsseite auf Suchen, um die Suche zu erzwingen
  • Sehen Sie sich das Log jeewhatsapp an: der Daemon zeigt Gruppe "xxx" nicht gefunden mit dem gesuchten Namen
Nachrichten werden nicht empfangen
  • Prüfen Sie, ob der Status im Tab WhatsApp-Verbindung Verbunden anzeigt
  • Prüfen Sie, ob das Feld Verknüpfte Gruppe ausgefüllt ist
  • Nur Textnachrichten der Kanalgruppe werden verarbeitet — Medien werden ignoriert
  • Sehen Sie sich das Log jeewhatsapp an, um zu prüfen, ob der Daemon die Nachrichten empfängt
Das Senden schlägt fehl
  • Prüfen Sie, ob der WhatsApp-Status Verbunden ist
  • Prüfen Sie, ob das Feld Verknüpfte Gruppe ausgefüllt ist
  • Testen Sie über den Tab Test (lassen Sie Empfänger leer, um in die Gruppe zu senden)
  • Sehen Sie sich das Log jeewhatsapp an
WhatsApp hat die Sitzung getrennt (Logout)
  • Der Daemon erkennt die Trennung und löscht die Zugangsdaten automatisch
  • Ein neuer QR-Code erscheint im Tab WhatsApp-Verbindung
  • Scannen Sie den QR-Code erneut — bei der Wiederverbindung wird die Kanalgruppe automatisch wiedergefunden

Technische Architektur

WhatsApp-Gruppe "jeewhatsapp" │ │══════════ Baileys WebSocket (direkte Verbindung) ══════════│ jeewhatsappd.js │ │── messages.upsert ──────────────────────────────────────▶ │ │ ├─ Filter: remoteJid === groupJid ? → sonst ignoriert │ ├─ Filter: fromMe ? → ignoriert (Nachricht von Jeedom) │ └─ POST callback.php?apikey= │ └─ jeewhatsapp::callback() │ └─ cmd.event(last_message, last_sender…) │ │◀── sock.sendMessage(groupJid, { text: Präfix + message }) │ POST 127.0.0.1:55148/action {"action":"send",…} │ ← Szenario → cmd::execute() → sendToDaemon()

Komponenten

KomponenteTechnologieRolle
jeewhatsappd.js Node.js (ESM) Daemon — Baileys-Verbindung, Kanalgruppen-Filter, lokaler HTTP-Server Port 55148
jeewhatsapp.class.php PHP Jeedom-Logik: Daemon-Lebenszyklus, Befehle, Gruppenversand
callback.php PHP Lokaler HTTP-Endpunkt, der vom Daemon bei jeder in der Gruppe empfangenen Nachricht aufgerufen wird
jeewhatsapp.ajax.php PHP AJAX-Aktionen: Testversand, QR-Code, findGroup, createGroup, Status
