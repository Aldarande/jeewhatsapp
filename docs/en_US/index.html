JeeWhatsApp Self-hosted

Two-way WhatsApp integration for Jeedom via Baileys.
A dedicated communication channel — 100 % local, no data on any third party.

Overview

JeeWhatsApp connects your Jeedom installation to WhatsApp via Baileys, an open-source library that establishes a direct WebSocket connection with WhatsApp Web. No third-party account, no API key, no subscription — everything happens between your Jeedom server and the WhatsApp servers.

⚠️ Unofficial application — risk of being banned JeeWhatsApp uses Baileys, a library that emulates WhatsApp Web. This plugin is not affiliated with Meta and does not use the official WhatsApp Business API. Excessive or robotic usage can lead to the temporary or permanent suspension of the WhatsApp account used, by Meta.

✅ Low-risk uses (recommended):
  • Sending a few home-automation alerts per day to a private group (motion, temperature, door opening…)
  • Replying to your own messages in the channel group via interactions
  • Using the plugin on a personal WhatsApp account that has been active for a long time
  • Sending messages to a group you belong to, with known contacts
❌ Uses to avoid:
  • Sending hundreds of messages per day automatically
  • Sending messages to strangers or to numbers not saved in your contacts
  • Using a recent WhatsApp account with no history of human conversations
  • Sending messages with perfectly regular timing (every X seconds)

What the plugin does not do (currently)

  • Sending media (images, videos, documents, voice messages)
  • Receiving messages outside the channel group
  • Sending to a number not registered on WhatsApp
  • Push notifications for the account owner — see limitation below
⚠️ Limitation — push notifications JeeWhatsApp uses your own WhatsApp account (linked via QR code). WhatsApp never notifies the account owner about their own messages — this rule also applies to mentions (@you), tested and confirmed to have no effect.

Consequence: when Jeedom posts in the channel group, you receive no push notification. Messages are visible by opening the group, but there is no sound alert or banner.

Solution: a second WhatsApp account dedicated to Jeedom. A virtual number (Google Voice, a cheap eSIM…) connected as the "Jeedom bot", a member of the channel group — your main account then receives the messages like any other group message, with normal notifications.

How it works — the channel group

JeeWhatsApp relies on a dedicated WhatsApp group that acts as a two-way communication channel between Jeedom and you.

🏠
Jeedom
⇄channel group
👥
WhatsApp group
"jeewhatsapp"
⇄notifications
📱
Your phone
  • Jeedom → group: every alert or report sent from a scenario appears in the group, prefixed (e.g. 🏠 Motion detected!)
  • Group → Jeedom: your messages in the group are received by Jeedom and trigger your scenarios or the interactions engine
  • Messages outside the group (direct messages, other groups) are ignored
  • You receive a WhatsApp notification just like for any other group message
💡 Jeedom prefix All messages sent by Jeedom are automatically prefixed (e.g. 🏠 ). This lets you visually distinguish Jeedom alerts from your own messages in the group. The daemon ignores its own messages (fromMe) to avoid loops.

Prerequisites

  • Jeedom 4.4 or higher
  • Node.js 18 or higher installed on the Jeedom machine
  • A smartphone with the WhatsApp app to scan the QR code
ℹ️ Check Node.js In a terminal on your Jeedom machine: node --version
The displayed version must be v18.x or higher.

Installation

  1. 1
    Install the plugin
    Copy the jeewhatsapp/ folder into the plugins/ directory of your Jeedom, then enable it in Plugins → Plugin management.
  2. 2
    Install the dependencies
    On the plugin page, click Install dependencies. The script installs @whiskeysockets/baileys, qrcode and pino via npm.
    This step can take 2 to 5 minutes depending on your internet connection.
  3. 3
    Start the daemon
    On the plugin configuration page, click Start the daemon. The status should turn to OK within a few seconds.
  4. 4
    Create a device and scan the QR code
    Create a device, configure the channel group, save, then go to the WhatsApp connection tab.

Device configuration

Go to Plugins → Communication → JeeWhatsApp and click Add.

FieldDescription
NameName shown in Jeedom (e.g. My WhatsApp)

Linked groupGroup JID — filled automatically, read-only
Jeedom interactionsEnables automatic replies via the interactions engine
Jeedom prefixText added at the start of every message sent by Jeedom (default: 🏠 )
Contact bookNamed aliases (Name=Number) — use a name as recipient instead of a phone number

Save. The commands are created automatically.

Link the channel group

After saving, you must link a WhatsApp group to the device. You must first be connected to WhatsApp (QR code scanned).

Option A — Existing group

  1. 1
    Create a WhatsApp group from your phone and name it (e.g. jeewhatsapp).
  2. 2
    On the device page, enter this name in the Channel group field.
  3. 3
    Click Search — the Linked group field is filled automatically with the JID.
  4. 4
    Save the device.

Option B — Create the group from Jeedom

  1. 1
    Enter the desired name in the Channel group field.
  2. 2
    Click Create — the group is created on WhatsApp and the JID is filled in.
  3. 3
    Save the device.
  4. 4
    From your phone, add the desired members to the group.
ℹ️ The "Linked group" field (JID) This read-only field contains the technical identifier of the WhatsApp group (format 120363…@g.us). It is filled automatically and must not be edited manually. The daemon stores it in auth/{id}/group_jid.txt to speed up restarts.
💡 Automatic search at startup As soon as the WhatsApp connection is established, the daemon automatically searches for the group by its name. The result appears in the jeewhatsapp logs: ✓ Group "jeewhatsapp" → 120363…@g.us.

QR code connection

Go to the WhatsApp connection tab on the device page.

  1. 1
    A QR code is displayed automatically (refreshed every 8 seconds). If the daemon was just started, wait a few seconds.
  2. 2
    Open WhatsApp on your phone → Settings → Linked devices → Link a device.
  3. 3
    Scan the QR code shown in Jeedom.
  4. 4
    The status turns to ✅ Connected. The daemon then automatically searches for the channel group.
💡 Persistent session Once connected, the credentials are saved in resources/jeewhatsappd/auth/{id}/. You will not need to rescan the QR code at each daemon restart.
ℹ️ Disconnection by WhatsApp If WhatsApp revokes access (rare), the session is deleted automatically and a new QR code appears in the tab. Simply rescan it.

Commands

INFO commands (read)

NamelogicalIdTypeDescription
Last message last_message string Text of the last message received in the channel group
Sender last_sender string Number of the group member who sent the message
Sender name last_sender_name string WhatsApp name of the sender
Received at last_received_at string Timestamp of the last message (YYYY-MM-DD HH:MM:SS)
Sent (hour) sent_hour numeric Number of messages sent during the current hour (logged)

ACTION commands

NamelogicalIdSubtypeDescription
Send a message send_message message Sends a message to the channel group.
Title = optional override (empty = channel group, number or contact book alias = direct send).
Message = text to send.
Reply reply message Sends the message to the channel group. The reply is visible to all members.
💡 "Title" field — optional override Jeedom shows two fields for message-type commands: Title and Message.
In JeeWhatsApp, the Title field is an optional override:
— Empty → send to the channel group (default behavior)
— Number (e.g. 33612345678) → direct send to that number
— Group JID (e.g. 120363…@g.us) → send to that specific group
— Alias (e.g. Didier) → resolved via the contact book

Jeedom interactions

When the Jeedom interactions option is enabled, every message received in the channel group is submitted to the Jeedom interactions engine. If an interaction matches, the reply is automatically sent to the channel group.

Message received in the groupJeedom's automatic reply
living room temperature🏠 The living room temperature is 21°C
turn on the light🏠 Light turned on
alarm status🏠 Alarm disabled
ℹ️ Configure interactions Interactions are configured in Tools → Interactions in Jeedom. They work with the commands of all your devices.
⚠️ Public replies in the group Automatic replies are sent to the group, visible to all members. If the group contains several people, everyone will see Jeedom's replies.

Message templates

The Templates tab lets you define reusable messages with Jeedom variables. Rather than rewriting the same text in each scenario, you define a template once and call it by its key.

Format

One template per line, in the format key=Message text:

welcome=Welcome home!
alert=⚠️ Alert: #[Home][Detector][Presence]# !
night=Good night — automatic shutter closing
report=☀️ Living room: #[Living room][Temperature][Value]#°C — Door: #[Entrance][Opening][Value]#
# Lines starting with # are comments
  • The template key is case-insensitive (Alert = alert)
  • The Jeedom tags #[Object][Device][Command]# are resolved automatically when sending. ⚠️ The object, device and command names are case-sensitive (as everywhere in Jeedom): they must match exactly — #[Living room][Temperature][Value]# ≠ #[living room][temperature][value]#
  • Empty lines and those starting with # are ignored

Use in a scenario

Use the Send a template action command and enter the key in the Message field:

📋 Send a template from a scenario

Action[My WhatsApp][Send a template]
Messagealert (template key)
Title(empty = channel group)
💡 Preview without saving The Preview templates button in the tab shows the list of recognized templates in real time, without having to save — handy to validate the syntax before recording.
ℹ️ Difference from Send a message Send a message sends the entered text directly.
Send a template first resolves the key to retrieve the template text, then resolves the Jeedom tags before sending. This lets you centralize your messages and edit them in a single place without touching the scenarios.

Contact book

The Contact book field lets you define named aliases for phone numbers. Use a name as the recipient in the send_message command instead of a raw phone number.

Format

One alias per line, in the format Name=Number:

Didier=33612345678
Marie=33698765432
Dad=33607080910
# Lines starting with # are comments
  • The name is case-insensitive (Didier = didier)
  • The number must be in international format without the + (e.g. 33612345678)
  • Empty lines and those starting with # are ignored

Use in a scenario

📱 Send a direct message to a contact by name

Action[My WhatsApp][Send a message]
TitleDidier (contact book alias)
MessageMotion detected at home!
💡 Autocomplete in the test interface In the Test tab, the Recipient field shows your contact book as suggestions: the name appears in bold, the number in parentheses as a hint.

Troubleshooting

I don't receive a WhatsApp notification when Jeedom sends a message
  • This is a WhatsApp limitation: an account never receives a notification for its own messages
  • Mentions @you do not bypass this rule — tested and confirmed
  • Only solution: use a second WhatsApp account dedicated to Jeedom (virtual number or dedicated SIM), a member of the channel group
  • Your main account then receives Jeedom's messages like any other group message, with normal notifications
The daemon does not start
  • Check that Node.js 18+ is installed: node --version
  • Re-run the dependency installation from the plugin page
  • Check the jeewhatsapp log in Analysis → Logs
  • Check that port 55148 (default port) is not already used by another service — if so, change it in Plugins → JeeWhatsApp → Configuration
The QR code does not appear
  • Check that the daemon is started (status OK in plugin management)
  • Restart the daemon then refresh the device page
  • Check the jeewhatsapp log to detect a Baileys error
  • If a session already exists, the QR code does not appear — the status must be Connected
The channel group is not found
  • Check that the name in Channel group matches exactly the name of the WhatsApp group (case-sensitive)
  • The group must exist on WhatsApp and your account must be a member of it
  • Click Search on the configuration page to force the search
  • Check the jeewhatsapp log: the daemon shows Group "xxx" not found with the searched name
Messages are not received
  • Check that the status in the WhatsApp connection tab shows Connected
  • Check that the Linked group field is filled in
  • Only text messages from the channel group are processed — media are ignored
  • Check the jeewhatsapp log to see whether the daemon is receiving the messages
Sending fails
  • Check that the WhatsApp status is Connected
  • Check that the Linked group field is filled in
  • Test from the Test tab (leave Recipient empty to send to the group)
  • Check the jeewhatsapp log
WhatsApp disconnected the session (logout)
  • The daemon detects the disconnection and deletes the credentials automatically
  • A new QR code appears in the WhatsApp connection tab
  • Rescan the QR code — on reconnection, the channel group is found automatically

Technical architecture

WhatsApp group "jeewhatsapp" │ │══════════ Baileys WebSocket (direct connection) ══════════│ jeewhatsappd.js │ │── messages.upsert ──────────────────────────────────────▶ │ │ ├─ filter: remoteJid === groupJid ? → otherwise ignored │ ├─ filter: fromMe ? → ignored (message from Jeedom) │ └─ POST callback.php?apikey= │ └─ jeewhatsapp::callback() │ └─ cmd.event(last_message, last_sender…) │ │◀── sock.sendMessage(groupJid, { text: prefix + message }) │ POST 127.0.0.1:55148/action {"action":"send",…} │ ← scenario → cmd::execute() → sendToDaemon()

Components

ComponentTechnologyRole
jeewhatsappd.js Node.js (ESM) Daemon — Baileys connection, channel group filter, local HTTP server port 55148
jeewhatsapp.class.php PHP Jeedom logic: daemon lifecycle, commands, group sending
callback.php PHP Local HTTP endpoint called by the daemon for every message received in the group
jeewhatsapp.ajax.php PHP AJAX actions: test send, QR code, findGroup, createGroup, status

FileContent
